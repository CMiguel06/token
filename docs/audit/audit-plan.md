# Plano de auditoria

## Verificações mínimas

- confirmar unicidade de token commitment;
- confirmar sequência contínua do bulletin board;
- recalcular hashes dos boletins;
- recalcular Merkle root;
- validar proof logs;
- verificar apuramento contra boletins publicados;
- gerar relatório de risco;
- documentar achados.

## Evidência

Cada achado deve conter:

```txt
id
severidade
componente afectado
evidência técnica
regra violada
recomendação
estado
```
