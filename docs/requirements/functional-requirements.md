# Requisitos funcionais

| ID | Requisito |
|---|---|
| FR-01 | O sistema deve permitir criar uma eleição com parâmetros públicos. |
| FR-02 | O sistema deve emitir uma credencial de voto apenas para eleitores elegíveis. |
| FR-03 | O sistema deve separar a verificação de identidade da submissão do boletim. |
| FR-04 | O sistema deve aceitar apenas boletins cifrados. |
| FR-05 | O sistema deve rejeitar a reutilização de uma credencial já gasta. |
| FR-06 | O sistema deve publicar boletins aceites num bulletin board append-only. |
| FR-07 | O sistema deve calcular uma Merkle root para prova de integridade pública. |
| FR-08 | O sistema deve permitir auditoria independente de sequência, unicidade e integridade. |
| FR-09 | O sistema deve produzir proof logs para cada verificação formal relevante. |
| FR-10 | O sistema deve gerar relatórios de risco de auditoria, sem alterar o apuramento. |
